Question 2
What are the required parameters, and what do they represent?

SERVICE=WMS ... service type.

REQUEST=GetCapabilities ... request name.

Other parameters are optional.

What is returned in response to a WMS GetCapabilities request?

This is from the esri site: The GetCapabilities request provides the following information about a WMS service:

All interfaces a WMS service can support

Image formats it can serve (e.g. jpeg, png, gif)

List of spatial reference systems available for delivery of map data

List of all exception formats for return of exceptions

List of all vendor specific capabilities (or properties) that are available for modifying or controlling actions of the service with the current value of each capability

List of one or more map layers available in the service

List of layers supporting the optional GetFeatureInfo interface

Question 3

What are the required parameters, and what do they represent?

VERSION=version ... Request version.

REQUEST=GetMap ... Request name.

LAYERS=layer_list ... Comma-separated list of one or more map layers.

STYLES=style_list ... Comma-separated list of one rendering style per requested layer. Optional if SLD parameter is present.

SRS=namespace:identifier ... Spatial Reference System.

CRS=namespace:identifier ... Spatial Reference System.

BBOX=minx,miny,maxx,maxy ... Bounding box corners (lower left, upper right)in SRS units.

WIDTH=output_width ... 1R0 Width in pixels of map picture.

What is returned in response to a WMS GetMap request?

From esri: "A GetMap request retrieves a map from a WMS site. GetMap allows the client to specify distinct layers, the spatial reference system, the geographic area, and other parameters describing the returned map format."

What is the significance of transparency in WMS requests?

From esri: It provides a "Map’s background transparency. Default is FALSE."
